# План разработки программы транскрибатора

Ниже представлен пошаговый план создания программы, которая использует модели **Vosk** и **Whisper** для преобразования аудио в текст.

## 1. Требования

- Python 3.8+
- [Vosk](https://alphacephei.com/vosk/)
- Пакет `whisper` (или альтернативный open-source вариант Whisper)
- Библиотеки для работы с аудио: `pydub`, `soundfile`

## 2. Структура проекта

```
transkriptorEXE/
├── transcribers/
│   ├── __init__.py
│   ├── vosk_transcriber.py
│   └── whisper_transcriber.py
├── utils/
│   └── audio.py
├── main.py
└── README.md
```

- **transcribers/** – модули с реализацией Vosk и Whisper транскрипторов.
- **utils/** – вспомогательные утилиты (например, загрузка и конвертация аудио).
- **main.py** – CLI-интерфейс для запуска программы.

## 3. Основные шаги реализации

1. **Инициализация проекта**: подготовить структуру каталогов и файл `setup.py` (опционально).
2. **Модуль загрузки и подготовки аудио**: конвертация форматов, нарезка по чанкам, приведение к нужной частоте дискретизации.
3. **Реализация VoskTranscriber**:
   - Загрузка офлайн-модели Vosk.
   - Распознавание аудио из файла или микрофона.
4. **Реализация WhisperTranscriber**:
   - Поддержка выбора модели (base, small, medium и т.д.).
   - Обработка длинных файлов через нарезку и объединение результатов.
5. **Общий интерфейс**: класс `Transcriber`, который абстрагирует работу с Vosk или Whisper.
6. **CLI**: парсер аргументов командной строки, позволяющий выбрать движок распознавания и дополнительные параметры (язык, модель, путь к файлу и т.д.).
7. **Тестирование**: базовые unit-тесты для утилит и транскрипторов.
8. **Сборка standalone-версии (опционально)**: использование `pyinstaller` для создания исполняемого файла.

## 4. Пример использования CLI

```bash
python main.py --engine vosk --model-path models/vosk-model "audio.wav"
python main.py --engine whisper --model small "audio.wav"
```
